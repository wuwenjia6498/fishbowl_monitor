## 🔄 如何查看修改效果

### 方法 1：强制刷新浏览器（最快）⚡

在浏览器中按下：

- **Windows/Linux**: `Ctrl + Shift + R` 或 `Ctrl + F5`
- **Mac**: `Cmd + Shift + R`

这会清除缓存并重新加载页面。

---

### 方法 2：清除浏览器缓存

1. 打开浏览器开发者工具（F12）
2. 右键点击刷新按钮
3. 选择"清空缓存并硬性重新加载"

---

### 方法 3：重启开发服务器（如果上述方法无效）

在终端中：

```bash
# 停止当前服务器（Ctrl + C）
# 然后重新启动
npm run dev
```

---

## 🎯 如何验证修改是否生效

### 1. 检查列宽

**修改前：**
- "指数名称"列可能会很宽，挤占其他列的空间

**修改后：**
- "指数名称"列固定为 140px 宽度
- 其他数据列（现价、MA20、状态等）获得更多空间
- 表格布局更均衡

### 2. 检查浏览器控制台

打开开发者工具（F12），查看：

1. **Elements 标签**：
   - 找到表格的 `<th>` 元素
   - 查看是否有 `w-[140px] min-w-[140px]` 类名
   - 检查计算后的样式是否为 `width: 140px`

2. **Console 标签**：
   - 确认没有 JavaScript 错误
   - 确认没有样式加载失败

## ❓ 如果仍然看不到变化

### 检查清单：

- [ ] 是否强制刷新了浏览器？（Ctrl+Shift+R）
- [ ] 浏览器控制台是否有错误？
- [ ] 开发服务器是否正常运行？（终端显示 Ready）
- [ ] 是否在正确的页面？（http://localhost:3000）
- [ ] 是否清除了浏览器缓存？

### 终极方案：完全重启

```bash
# 1. 停止开发服务器（终端3，按 Ctrl+C）
# 2. 清除 Next.js 缓存
rm -rf .next
# Windows PowerShell:
Remove-Item -Recurse -Force .next

# 3. 重新启动
npm run dev
```
